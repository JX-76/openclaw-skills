# Funnel Analysis - 漏斗分析

---

## Description

分析用户转化漏斗，找出流失环节和优化机会。

**解决的实际问题：**
- 整体转化率低，不知道哪里出问题
- 优化了A环节，B环节掉了
- 不知道流失用户的特征

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"漏斗分析"、"转化分析"
- 整体转化率 < 20%
- 有明确的多步骤流程
- 需要优化转化率

**不适用场景：**
- 单步骤产品（如计算器）
- 用户路径极度分散

---

## Pre-conditions

**必须提供：**
- 漏斗步骤定义（明确的起点和终点）
- 各步骤事件数据
- 时间范围

---

## Workflow

### Phase 1: 定义漏斗

```markdown
## 漏斗定义

### 核心漏斗（以电商为例）
| 步骤 | 事件 | 说明 |
|------|------|------|
| 1 | 浏览商品 | 进入商品详情页 |
| 2 | 加入购物车 | 点击「加入购物车」 |
| 3 | 进入结算 | 点击「去结算」 |
| 4 | 提交订单 | 点击「提交订单」 |
| 5 | 完成支付 | 支付成功 |

### 关键原则
- 步骤必须是用户主动行为（非被动曝光）
- 步骤之间无遗漏（MECE原则）
- 终点是明确的价值点（如付费）

### 本次分析定义
- 产品：记账App
- 漏斗：启动App → 点击记账 → 完成记账 → 次日返回
```

### Phase 2: 计算转化率

```markdown
## 转化数据

| 步骤 | 用户数 | 转化率 | 累计转化 |
|------|--------|--------|---------|
| 启动App | 10,000 | 100% | 100% |
| 点击记账 | 6,000 | 60% | 60% |
| 完成记账 | 4,500 | 75% | 45% |
| 次日返回 | 3,600 | 80% | 36% |

### 关键指标
- 整体转化率：36%（启动→次日留存）
- 最大流失：启动→点击（流失40%）
- 最顺畅：完成→次日（留存80%）
```

### Phase 3: 下钻分析

```markdown
## 流失分析

### Step 1→2 流失分析（40%流失）
**可能原因**：
- 首页引导不足（用户不知道点哪）
- 加载慢，用户等待中退出
- 推送干扰，用户被打断

**验证方法**：
- 热力图看首页点击分布
- 页面加载时长分析
- 流失时间点分析

### Step 2→3 流失分析（25%流失）
**可能原因**：
- 记账流程复杂（步骤多）
- 分类选择困难
- 键盘体验差

**优化方案**：
- 简化流程（3步→2步）
- 智能推荐分类
- 优化键盘交互
```

### Phase 4: 细分对比

```markdown
## 漏斗对比

### 按渠道
| 渠道 | 启动→点击 | 点击→完成 | 完成→次日 |
|------|----------|----------|----------|
| 自然 | 65% | 78% | 82% |
| 付费 | 55% | 70% | 75% |

**结论**：付费渠道用户质量差，需优化投放

### 按版本
| 版本 | 整体转化 |
|------|---------|
| v1.0 | 32% |
| v1.1 | 36% |
| v1.2 | 34% |

**结论**：v1.2优化后反而下降，需回滚检查

### 按设备
| 设备 | 启动→点击 |
|------|----------|
| iOS | 65% |
| Android | 55% |

**结论**：Android端问题大，需专项优化
```
