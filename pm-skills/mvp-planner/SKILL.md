# MVP Planner - MVP规划

---

## Description

将大需求拆解为最小可行产品，快速验证核心价值。

**解决的实际问题：**
- 需求越做越大，迟迟无法上线
- 想一次性做完美，结果错过市场窗口
- 不确定哪些功能可以砍掉

---

## When to Use

**强制触发条件（任一满足）：**
- 用户明确说"MVP"、"最小版本"、"先上一版"
- 需求范围过大，开发周期 > 1个月
- 需要快速验证市场

---

## Pre-conditions

**必须提供：**
- 完整需求描述
- 核心目标（MVP要验证什么）

---

## Workflow

### Phase 1: 核心价值识别

**追问本质：**
```
完整需求：做一个电商会员系统，包含等级、积分、权益、营销工具

核心问题：没有会员系统，用户现在怎么解决？
→ 用户流失，无复购激励

MVP核心：给用户一个"回头买的理由"
```

### Phase 2: 功能拆解与分类

```markdown
## 功能清单

### 必须有（MVP）
- [x] 会员等级（3级）
- [x] 基础积分（消费得积分）
- [x] 积分抵现（最直观的价值）

### 应该有（V1.1）
- [ ] 会员专属价
- [ ] 积分商城
- [ ] 升级提醒

### 可以有（V1.2+）
- [ ] 储值功能
- [ ] 会员营销工具
- [ ] 数据分析看板
```

**MVP判定标准：**
- 没有这个功能，核心价值是否还存在？
- 手动/临时方案能否替代？
- 开发成本是否可控（<2周）？

### Phase 3: 流程简化

**简化前：**
```
用户注册 → 完善资料 → 首单 → 积分到账 → 查看积分 → 积分兑换 → 使用抵现
```

**MVP简化：**
```
首单 → 自动积分 → 下次下单自动抵现
（砍掉：注册引导、积分明细页、兑换页）
```

### Phase 4: 验证指标

```markdown
## MVP成功指标

### 必须达成（否则MVP失败）
- 会员注册率 > 20%（访问用户）
- 复购率提升 > 5%（对比非会员）

### 建议达成
- 积分使用率 > 30%
- 客单价提升 > 10%

### 观察指标
- 功能使用路径
- 用户反馈关键词
```

---

## Output

```markdown
# MVP规划文档

## 核心目标
验证：会员积分系统能否提升复购率

## MVP范围
### 包含功能
1. 自动积分（消费金额1:1）
2. 自动抵现（100积分=1元）
3. 简单等级（根据累计消费自动升级）

### 不包含功能
- 积分明细查询
- 积分兑换商品
- 会员中心页面
- 营销触达

## 用户流程
[简化流程图]

## 验证计划
- 上线周期：2周
- 灰度范围：10%用户
- 观察周期：2周
- 决策点：复购率是否提升5%？

## 后续规划
- 达标：继续完善会员中心、增加权益
- 不达标：分析原因，调整策略或放弃
```
