# Monetization Model - 商业模式设计

---

## Description

设计可持续的商业模式，明确收入来源和盈利路径。

**解决的实际问题：**
- 有用户但不赚钱
- 不知道有哪些变现方式
- 变现影响用户体验

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"商业模式"、"怎么赚钱"、"变现"
- 产品验证完成，准备规模化
- 融资需要商业计划书
- 现有模式收入停滞

**不适用场景：**
- 产品未验证（先验证价值）
- 纯公益项目

---

## Pre-conditions

**必须提供：**
- 用户画像和使用场景
- 当前成本结构
- 竞品商业模式

---

## Workflow

### Phase 1: 变现方式盘点

```markdown
## 变现方式矩阵

| 方式 | 适用产品 | 优点 | 缺点 |
|------|---------|------|------|
| 订阅 | 工具/SaaS | 稳定收入 | 用户抗拒 |
| 广告 | 内容/工具 | 免费获客 | 影响体验 |
| 交易抽成 | 平台 | 规模效应 | 需交易量 |
| 数据服务 | B端 | 高毛利 | 合规风险 |
| 增值服务 | 游戏/工具 | 灵活 | 设计复杂 |

### 记账App可选模式
1. **订阅**：高级功能（云同步/导出）
2. **广告**：免费用户展示广告
3. **联盟**：推荐理财产品抽佣
4. **数据**：匿名消费趋势报告（B端）
```

### Phase 2: 设计商业模式

```markdown
## 商业模式画布

| 要素 | 内容 |
|------|------|
| 价值主张 | 最快记账体验 |
| 客户细分 | 25-35岁职场白领 |
| 渠道 | 应用商店/口碑 |
| 收入流 | 订阅+广告+联盟 |
| 成本结构 | 开发+服务器+市场 |

### 收入结构预测
| 来源 | 占比 | 说明 |
|------|------|------|
| 订阅 | 60% | 核心收入 |
| 广告 | 25% | 免费用户 |
| 联盟 | 15% | 理财产品推荐 |
```

### Phase 3: 验证与迭代

```markdown
## 验证计划

### 阶段1：单一变现（MVP）
- 仅订阅，验证付费意愿
- 目标：付费率 > 2%

### 阶段2：组合变现
- 加入广告和联盟
- 监控用户体验指标

### 阶段3：优化结构
- 根据数据调整占比
- 探索B端数据服务
```
