# AI UX Design - AI 交互设计

---

## Description

设计 AI 功能的人机交互，平衡智能与可控。

**解决的实际问题：**
- AI 输出不确定，用户不知道怎么用
- AI 出错用户不信任
- 等待 AI 响应体验差

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"AI 交互"、"AI 体验设计"
- AI 功能有可见的用户界面
- AI 输出需要用户确认/修改
- AI 有延迟（>500ms）

**不适用场景：**
- 纯后台 AI（用户无感知）
- 实时性要求极高的场景

---

## Pre-conditions

**必须提供：**
- AI 功能场景
- 用户任务流程
- AI 输出类型

**可选提供：**
- 竞品 AI 交互参考
- 用户调研结果

## Workflow

### Phase 1: 交互模式选择

```markdown
## AI 交互模式

### 模式对比
| 模式 | 说明 | 适用场景 | 示例 |
|------|------|---------|------|
| 自动执行 | AI 直接操作，无确认 | 低风险+高确定性 | 智能分类 |
| 推荐确认 | AI 推荐，用户一键确认 | 中等风险 | 文案推荐 |
| 协作编辑 | AI 生成，用户修改 | 创意类 | 写作助手 |
| 问答式 | 用户提问，AI 回答 | 信息获取 | 智能客服 |
| 渐进式 | 从辅助到自动 | 建立信任 | 智能输入 |

### 本次选择
场景：智能记账分类
推荐模式：**推荐确认**
原因：
- 分错类影响用户体验（不可完全自动）
- 用户确认成本低（一键操作）
- 逐步建立信任后可改为自动
```

### Phase 2: 状态与反馈设计

```markdown
## AI 状态设计

### 1. 加载状态
```
AI 思考中...
[进度条动画]
预计 2 秒

优化：
- 显示具体动作（"正在分析商户类型..."）
- 超过 3 秒显示取消按钮
- 提供骨架屏，不空白等待
```

### 2. 成功状态
```
✅ 智能分类：餐饮
（基于：星巴克 → 咖啡 → 餐饮）

优化：
- 展示推理依据（建立信任）
- 提供修改入口
- 可关闭智能推荐
```

### 3. 不确定状态
```
🤔 不确定这是什么类型
推荐：餐饮（置信度 60%）
[确认] [选择其他]

优化：
- 诚实告知不确定
- 降低推荐置信度显示
- 提供备选选项
```

### 4. 错误状态
```
❌ 分类失败，请手动选择
[餐饮] [交通] [购物] ...

优化：
- 不展示错误代码
- 快速提供手动选择
- 后台记录用于优化
```
```

### Phase 3: 信任与可控设计

```markdown
## 建立信任

### 1. 透明度
- 告诉用户这是 AI 结果
- 展示推理依据
- 提供反馈渠道

### 2. 可控性
- 用户可随时修改
- 可关闭 AI 功能
- 可调整 AI 敏感度

### 3. 渐进式
```
阶段1：AI 后台推荐，不显示（观察准确率）
阶段2：显示推荐，默认不采用（建立习惯）
阶段3：显示推荐，默认采用，可修改（信任建立）
阶段4：自动采用，仅异常提示（高信任）
```

## 错误恢复
| 场景 | 设计 |
|------|------|
| AI 分错 | 一键修改 + 感谢反馈 |
| 连续出错 | 降级到手动 + 解释原因 |
| 用户不信任 | 提供纯手动模式 |
```

### Phase 4: 性能体验优化

```markdown
## 延迟处理

### < 500ms
- 直接展示结果
- 用户几乎无感知

### 500ms - 2s
- 显示加载动画
- 进度提示
- 骨架屏

### > 2s
- 提供预计时间
- 允许后台处理
- 完成后通知

### 预加载策略
- 用户输入时预调用 AI
- 本地缓存常见结果
- 渐进式展示（先显示大概，再细化）
```
