# Retention Analysis - 留存深度分析

---

## Description

深度分析用户留存，找到提升留存的关键杠杆。

**解决的实际问题：**
- 留存数字低，不知道怎么提升
- 不知道高留存用户有什么特征
- 留存和什么因素最相关

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"留存分析"、"怎么提升留存"
- 次日留存 < 30%
- 7日留存 < 15%
- 留存持续下降

**不适用场景：**
- 产品刚上线（<30天）
- 用户量<500

---

## Pre-conditions

**必须提供：**
- 留存数据（次日/7日/30日）
- 用户行为数据
- 用户画像数据

---

## Workflow

### Phase 1: 留存现状诊断

```markdown
## 留存曲线

| 时间 | 留存率 | 行业对比 |
|------|--------|---------|
| 次日 | 35% | 平均40% ❌ |
| 7日 | 18% | 平均20% ⚠️ |
| 30日 | 8% | 平均10% ⚠️ |

### 留存曲线类型
```
类型A（健康）：
100% → 45% → 35% → 30% → 28% → 27% （快速平稳）

类型B（问题）：
100% → 45% → 25% → 15% → 10% → 6% （持续下降）

类型C（我们的现状）：
100% → 35% → 18% → 12% → 8% → 6% （低于平均）
```

**结论**：次日留存是最大问题，首体验需优化
```

### Phase 3: 关键行为识别

```markdown
## Magic Number 分析

### 假设：完成X次记账后留存显著提升
| 完成次数 | 用户数 | 7日留存 | 提升 |
|----------|--------|---------|------|
| 1次 | 2000 | 15% | - |
| 2次 | 1500 | 22% | +7pp |
| 3次 | 1200 | 35% | +13pp ✅ |
| 5次 | 800 | 48% | +26pp |

### 结论
**Magic Number = 3次记账**
- 完成3次记账的用户，7日留存35%
- 比完成1次的提升20pp
- 是关键拐点

### 行动
- 新用户引导目标：3天内完成3笔记账
- 设计激励机制：完成3笔解锁高级功能
```

### Phase 4: 留存提升方案

```markdown
## 分层留存策略

### 新用户（0-3天）- 重点优化
**目标**：次日留存 35% → 45%
**措施**：
1. 简化首单流程（延迟注册）
2. 完成首单后立即展示统计（Aha时刻）
3. 推送提醒（次日）

### 早期用户（4-7天）- 习惯养成
**目标**：7日留存 18% → 25%
**措施**：
1. 连续记账奖励（3天/7天徽章）
2. 周报推送（展示成果）
3. 预算设置引导

### 中期用户（8-30天）- 深度使用
**目标**：30日留存 8% → 12%
**措施**：
1. 月度财务报告
2. 社区功能（记账小组）
3. 高级功能解锁

### 长期用户（30天+）- 防流失
**目标**：降低流失率
**措施**：
1. 流失预警（7天未记账）
2. 召回推送（"想你了"）
3. 年度回顾（数据可视化）
```

## Output

```markdown
# 留存分析报告

## 现状诊断
- 次日留存35%（低于行业平均40%）
- 留存曲线持续下降，未稳定

## 关键发现
- Magic Number：3次记账（留存从15%→35%）
- 高留存特征：设置预算、高收入、口碑推荐
- 最大机会：新用户首3天体验

## 提升方案
| 阶段 | 措施 | 目标 | 预期提升 |
|------|------|------|---------|
| 新用户 | 简化首单+立即价值展示 | 次日留存>45% | +10pp |
| 早期 | 连续记账激励 | 7日留存>25% | +7pp |
| 中期 | 月度报告+社区 | 30日留存>12% | +4pp |

## 验证计划
- A/B测试：对照组 vs 新用户引导优化组
- 观察指标：次日留存、3次记账完成率
- 时间：2周
```
