# Model Evaluation - 模型评估

---

## Description

系统评估 AI 模型表现，确保达到上线标准。

**解决的实际问题：**
- 模型准确率多少算合格？
- 不同模型怎么选？
- 线上表现和离线测试不一致

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"评估模型"、"模型测试"
- 模型上线前验收
- 对比多个模型选型
- 线上效果不达预期，需诊断

**不适用场景：**
- 直接使用第三方 API（无需评估）
- 原型阶段（仅需定性验证）

---

## Pre-conditions

**必须提供：**
- 测试数据集（标注好的）
- 评估指标定义
- 及格线标准

**可选提供：**
- 对比模型
- 线上 A/B 测试数据

---

## Workflow

### Phase 1: 指标定义

```markdown
## 评估指标

### 分类任务
| 指标 | 公式 | 适用场景 |
|------|------|---------|
| 准确率 | (TP+TN)/(TP+TN+FP+FN) | 类别均衡 |
| 精确率 | TP/(TP+FP) | 避免误杀（垃圾邮件） |
| 召回率 | TP/(TP+FN) | 避免漏检（医疗） |
| F1 | 2×精确率×召回率/(精确率+召回率) | 平衡两者 |

### 生成任务
| 指标 | 说明 | 工具 |
|------|------|------|
| BLEU | 和参考答案 n-gram 重合 | nltk |
| ROUGE | 召回导向的文本重叠 | rouge |
| 人工评分 | 1-5分主观评价 | - |

### 本次评估
任务：记账分类（10类）
指标：
- 主要：准确率（Top-1）
- 辅助：Top-3 准确率（正确分类在前3）
- 及格线：准确率 > 85%
- 优秀线：准确率 > 90%
```

### Phase 2: 数据集准备

```markdown
## 测试集构建

### 数据要求
- 数量：每类至少 100 条
- 分布：和线上分布一致
- 质量：人工标注，双人校验
- 时效：近 3 个月数据

### 数据集划分
```
训练集：70%（模型训练）
验证集：15%（调参）
测试集：15%（最终评估）
```

### 数据增强（测试鲁棒性）
- 拼写错误："星巴克"→"星巴客"
- 口语化："吃了个饭"→"刚吃了顿饭"
- 缩写："KFC"→"肯德基"
```

### Phase 3: 离线评估

```markdown
## 评估执行

### 1. 单模型评估
```python
from sklearn.metrics import accuracy_score, classification_report

# 预测
y_pred = model.predict(X_test)

# 指标
accuracy = accuracy_score(y_test, y_pred)
print(f"准确率: {accuracy:.2%}")

# 详细报告
print(classification_report(y_test, y_pred, target_names=categories))
```

### 2. 混淆矩阵分析
```
          餐饮  交通  购物  ...
餐饮      85    5    3   ...
交通       4   88    2   ...
购物       3    2   90   ...

分析：
- 餐饮↔交通混淆较多（可能都是外出场景）
- 购物准确率最高
```

### 3. 错误案例分析
| 输入 | 真实 | 预测 | 原因 |
|------|------|------|------|
| 地铁充值 | 交通 | 其他 | 未识别"地铁" |
| 超市买水 | 购物 | 餐饮 | 和餐饮场景混淆 |

### 4. 对比评估（多模型）
| 模型 | 准确率 | 延迟 | 成本 | 结论 |
|------|--------|------|------|------|
| BERT-base | 87% | 50ms | 低 | 及格 |
| BERT-large | 91% | 120ms | 中 | 推荐 |
| GPT-4 | 94% | 800ms | 高 | 贵 |
```

### Phase 4: 上线标准检查

```markdown
## 上线 Checklist

### 性能标准
- [ ] 准确率 ≥ 85%（及格线）
- [ ] Top-3 准确率 ≥ 95%
- [ ] P95 延迟 < 200ms
- [ ] 内存占用 < 500MB

### 鲁棒性
- [ ] 拼写错误处理正确率 > 80%
- [ ] 超长文本（>100字）不崩溃
- [ ] 空输入有默认处理

### 可维护性
- [ ] 有监控和告警
- [ ] 可快速回滚
- [ ] 有降级方案

## 评估结论
| 维度 | 结果 | 状态 |
|------|------|------|
| 准确率 | 87% | ✅ 及格 |
| 延迟 | 150ms | ✅ 达标 |
| 鲁棒性 | 良好 | ✅ 通过 |
| 综合 | - | ✅ 可上线 |
```
