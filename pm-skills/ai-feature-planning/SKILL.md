# AI Feature Planning - AI 功能规划

---

## Description

规划 AI 功能，找到适合用 AI 解决的产品场景。

**解决的实际问题：**
- 什么都想用 AI，不知道哪里真需要
- AI 功能成本高，不知道 ROI
- AI 出错影响用户体验

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"加个 AI 功能"、"用 AI 优化"
- 评估是否用 AI 解决某个问题
- 规划 AI 产品路线图
- 现有功能考虑 AI 升级

**不适用场景：**
- 规则就能解决的问题（无需 AI）
- 对准确率要求 100%（AI 会出错）
- 数据量太小（<1000 条）

---

## Pre-conditions

**必须提供：**
- 问题场景（要解决什么）
- 当前解决方案（如有）
- 数据情况（是否有训练数据）

**可选提供：**
- 竞品 AI 功能
- 技术可行性评估

## Workflow

### Phase 1: AI 适用性评估

```markdown
## AI 适用性检查

### 适合 AI 的场景
| 特征 | 示例 | 我们的场景 |
|------|------|-----------|
| 非结构化输入 | 文本/图片/语音 | ? |
| 难以规则化 | 语义理解/创意生成 | ? |
| 容忍一定错误 | 推荐/辅助决策 | ? |
| 数据量大 | 有历史数据积累 | ? |
| 人工成本高 | 客服/审核 | ? |

### 不适合 AI 的场景
- [ ] 规则明确（如计算/校验）
- [ ] 要求 100% 准确（如支付/医疗诊断）
- [ ] 数据极少（冷启动问题）
- [ ] 实时性要求极高（<100ms）

### 本次评估
场景：智能记账分类
- ✅ 非结构化：用户输入描述各异
- ✅ 难以规则：商户名千变万化
- ⚠️ 容错性：分错类可手动修改
- ✅ 数据量：10万+历史记账记录
- ✅ 人工成本：手动分类费时

结论：✅ 适合 AI
```

### Phase 2: 方案设计

```markdown
## AI 方案对比

| 方案 | 实现方式 | 准确率 | 成本 | 延迟 | 适用 |
|------|---------|--------|------|------|------|
| 规则 | 关键词匹配 | 60% | 低 | 快 | 简单场景 |
| 传统 ML | 训练分类模型 | 85% | 中 | 快 | 有标注数据 |
| LLM | 调用 GPT-4 | 90% | 高 | 慢 | 复杂理解 |
| 混合 | 规则+LLM兜底 | 88% | 中 | 中 | 平衡方案 |

### 推荐方案：混合方案
```
1. 规则层：常见商户直接匹配（占 60%）
2. ML 层：模型预测中等 confidence 的（占 30%）
3. LLM 层：复杂描述调用 GPT（占 10%）
```

### 降级策略
- AI 服务异常 → 回退到规则
- 用户不满意 → 提供手动修改
- 新商户无数据 → 标记为「其他」待学习
```

### Phase 3: 体验设计

```markdown
## AI 交互设计

### 1. 预期管理
```
用户输入："星巴克咖啡"
AI 分类："餐饮"
UI 显示："智能分类：餐饮 [修改]"
说明：让用户知道这是 AI 分类，可以改
```

### 2. 错误处理
```
场景：AI 分错类
设计：
- 允许一键修改
- 记录修正数据用于优化
- 感谢用户反馈（"已记录，会越来越准"）
```

### 3. 渐进式智能
```
阶段1：后台推荐，不自动填充（观察准确率）
阶段2：自动填充，但可修改（准确率>80%）
阶段3：自动填充，仅异常提示（准确率>95%）
```

### 4. 反馈闭环
```
数据飞轮：
用户记账 → AI 分类 → 用户修正 → 数据回流 → 模型优化
```
```

### Phase 4: 成本与 ROI 评估

```markdown
## 成本分析

### 直接成本
| 项目 | 估算 | 说明 |
|------|------|------|
| LLM 调用 | ¥0.005/次 | GPT-4 API |
| 模型训练 | ¥5000/次 | 初始+每月迭代 |
| 标注成本 | ¥0.5/条 | 人工标注 |

### 效益评估
- 用户节省：每次分类节省 3 秒
- 日活 1 万，日均记账 2 次
- 总节省时间：10000×2×3=60000 秒/天
- 用户价值：体验提升 → 留存 +5%

### ROI 计算
月成本：¥3000（API）+ ¥5000（训练）= ¥8000
预期收益：留存提升 5% → LTV 提升 ¥3/人
需要：¥8000/¥3 ≈ 2667 人提升即可回本
当前日活 1 万 → 5% = 500 人/天
结论：3 个月可回本 ✅
```
