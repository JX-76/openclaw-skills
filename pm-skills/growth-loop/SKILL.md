# Growth Loop - 增长飞轮设计

---

## Description

设计自驱动的增长飞轮，降低获客成本，实现可持续增长。

**解决的实际问题：**
- 获客成本越来越高，烧不起钱
- 增长靠投放，一停就掉
- 用户来了留不住，留存差

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"增长"、"增长飞轮"、"growth loop"
- 获客成本 > 用户生命周期价值 1/3
- 增长主要靠付费投放
- 产品有网络效应/UGC潜力

**不适用场景：**
- 纯工具类（无网络效应）
- 客单价极高（B2B大客户）
- 合规限制强的行业（金融/医疗）

---

## Pre-conditions

**必须提供：**
- 当前获客渠道及成本
- 用户行为数据（留存/分享率）
- 产品核心价值点

**可选提供：**
- 竞品增长策略
- 用户分享动机洞察

---

## Workflow

### Phase 1: 诊断现状

```markdown
## 增长现状诊断

### 获客渠道分析
| 渠道 | 成本 | 量级 | 质量 | 可持续性 |
|------|------|------|------|---------|
| 应用商店 | ¥5/人 | 100/天 | 中 | 依赖ASO |
| 信息流 | ¥20/人 | 500/天 | 低 | 烧钱 |
| 自然搜索 | ¥1/人 | 50/天 | 高 | 慢 |

### 关键指标
- 获客成本（CAC）：¥15
- 生命周期价值（LTV）：¥30
- LTV/CAC = 2（健康但可优化）
- 病毒系数（K）：0.3（<1，未自增长）
```

### Phase 2: 识别增长杠杆

```markdown
## 增长杠杆分析

### 用户旅程中的增长机会
| 阶段 | 用户行为 | 增长机会 |
|------|---------|---------|
| 首次使用 | 完成核心功能 | 新用户激活 → 提升留存 |
| 使用中期 | 获得价值 | 触发分享 → 病毒传播 |
| 付费阶段 | 成为付费用户 | 提升ARPU |
| 深度使用 | 成为重度用户 | 邀请好友 → 裂变 |

### 可能的增长模式
1. **病毒式**（邀请好友得奖励）
2. **UGC**（用户内容吸引新用户）
3. **网络效应**（用户越多价值越大）
4. **SEO/内容**（内容带来自然流量）
```

### Phase 3: 设计增长飞轮

```markdown
## 增长飞轮设计（以记账App为例）

### 飞轮组件
```
用户记账 → 生成月度报告 → 分享报告到朋友圈 
    ↑                                      ↓
    新用户看到 → 下载App ← 好奇自己财务状况
```

### 飞轮增强点
| 环节 | 优化策略 | 预期提升 |
|------|---------|---------|
| 生成报告 | 自动美化图表，突出亮点 | 分享率 5%→15% |
| 分享引导 | 分享后解锁高级功能 | 分享率 15%→25% |
| 新用户落地 | 个性化首屏（"你的同龄人在XX上花了XX"）| 转化率 10%→20% |
| 激活流程 | 一键导入微信/支付宝账单 | 激活率 30%→50% |
```

### Phase 4: 实验与优化

```markdown
## 增长实验 backlog

### 高优先级（立即做）
| 实验 | 假设 | 指标 | 时间 |
|------|------|------|------|
| 分享报告解锁功能 | 激励提升分享率 | 分享率 | 2周 |
| 落地页个性化 | 提升新用户转化 | 注册率 | 2周 |

### 中优先级（验证后做）
| 实验 | 假设 | 指标 | 时间 |
|------|------|------|------|
| 排行榜功能 | 攀比心理驱动分享 | 分享率+留存 | 4周 |
| 记账小组 | 社交压力提升留存 | 7日留存 | 4周 |
```
