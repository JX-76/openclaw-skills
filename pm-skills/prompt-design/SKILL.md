# Prompt Design - Prompt 设计

---

## Description

设计高质量的 Prompt，让 AI 输出稳定、准确、可用的结果。

**解决的实际问题：**
- AI 输出不稳定，每次结果不一样
- 输出格式混乱，不好解析
- 模型理解偏差，答非所问

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"写个 prompt"、"优化 prompt"
- AI 功能输出不稳定
- 需要结构化输出（JSON/XML）
- 模型表现不达预期

**不适用场景：**
- 简单问答（无需复杂 prompt）
- 已封装好的 AI 组件

---

## Pre-conditions

**必须提供：**
- 任务目标（AI 要完成什么）
- 输入数据样例
- 期望输出格式

**可选提供：**
- 当前 prompt（如有）
- 失败案例

---

## Workflow

### Phase 1: 任务分析

```markdown
## 任务拆解

### 任务类型
| 类型 | 示例 | 设计重点 |
|------|------|---------|
| 分类 | 情感分析 | 类别定义清晰 |
| 生成 | 文案写作 | 风格+约束 |
| 提取 | 信息抽取 | 字段+格式 |
| 推理 | 逻辑判断 | 步骤+验证 |
| 转换 | 翻译/改写 | 保留+转换规则 |

### 本次任务
- 类型：提取
- 任务：从用户评论中提取产品优缺点
- 输入：用户评论文本
- 输出：JSON 格式 {"pros": [], "cons": []}
```

### Phase 2: Prompt 结构设计

```markdown
## Prompt 核心要素

### 1. 角色定义（Role）
```
你是一位专业的产品分析师，擅长从用户反馈中提取关键信息。
```

### 2. 任务说明（Task）
```
请分析以下用户评论，提取产品的优点和缺点。
```

### 3. 输入格式（Input）
```
用户评论：
{user_comment}
```

### 4. 输出格式（Output Format）
```
请以 JSON 格式输出，不要添加其他说明：
{
  "pros": ["优点1", "优点2"],
  "cons": ["缺点1", "缺点2"],
  "sentiment": "positive/negative/neutral"
}
```

### 5. 约束条件（Constraints）
```
- 优点和缺点各不超过3条
- 用简洁的语言描述
- 如果没有，返回空数组
```

### 6. 示例（Examples）
```
输入："这个 App 记账很快，但分类有点麻烦"
输出：
{
  "pros": ["记账速度快"],
  "cons": ["分类操作复杂"],
  "sentiment": "neutral"
}
```
```

### Phase 3: Prompt 优化技巧

```markdown
## 优化技巧

### 1. 少样本提示（Few-shot）
提供 2-3 个示例，让模型理解模式

### 2. 思维链（Chain-of-Thought）
要求模型展示推理过程
```
请一步一步分析：
1. 先理解评论整体意思
2. 找出正面描述
3. 找出负面描述
4. 总结情感倾向
```

### 3. 输出格式强制
使用特殊标记或强调
```
【必须】输出必须是合法 JSON
【禁止】不要添加 markdown 代码块标记
```

### 4. 边界情况处理
```
特殊情况：
- 评论为广告/垃圾信息 → 返回 {"error": "invalid"}
- 评论过短（<10字）→ 返回 {"error": "too_short"}
- 无明确优缺点 → 返回空数组
```

### 5. 温度调节
- 创意任务：temperature=0.7-0.9
- 精确任务：temperature=0.1-0.3
- 分类/提取：temperature=0
```

### Phase 4: 测试与迭代

```markdown
## 测试 checklist

### 功能测试
- [ ] 正常输入输出正确
- [ ] 边界输入处理合理
- [ ] 格式稳定（JSON 可解析）
- [ ] 多次运行结果一致

### 鲁棒性测试
- [ ] 输入为乱码
- [ ] 输入超长文本
- [ ] 输入非目标语言
- [ ] 输入包含特殊字符

### 性能测试
- [ ] 响应时间 < 2秒
- [ ] 输出长度合理

## 迭代记录
| 版本 | 修改点 | 效果 |
|------|--------|------|
| v1 | 初始版本 | 输出不稳定 |
| v2 | 添加 Few-shot | 准确率 70%→85% |
| v3 | 强制 JSON 格式 | 可解析率 80%→98% |
```
