# Unit Economics - 单位经济模型

---

## Description

计算单位用户经济模型，判断商业模式是否可持续。

**解决的实际问题：**
- 不知道每个用户赚多少/亏多少
- 增长越快亏越多
- 无法判断何时盈利

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"单位经济"、"LTV"、"CAC"、"ROI"
- 准备融资（投资人必问）
- 规模化前评估可行性
- 当前亏损，需判断方向

**不适用场景：**
- 产品早期（数据不足）
- 非商业项目

---

## Pre-conditions

**必须提供：**
- 获客成本（CAC）
- 用户留存数据
- ARPU（每用户平均收入）
- 变动成本

---

## Workflow

### Phase 1: 计算 LTV

```markdown
## LTV 计算方法

### 简化公式
LTV = ARPU × 用户平均生命周期

### 详细公式
LTV = ARPU × (1 / 月流失率)

### 记账App示例
- ARPU：¥6/月（付费用户¥12，免费¥0，平均付费率50%）
- 月流失率：10%
- 用户平均生命周期：10个月
- LTV = ¥6 × 10 = ¥60
```

### Phase 2: 计算 CAC

```markdown
## CAC 构成

| 成本项 | 金额 | 说明 |
|--------|------|------|
| 广告投放 | ¥8 | 主要获客渠道 |
| ASO优化 | ¥1 | 应用商店优化 |
| 内容营销 | ¥1 | 分摊到单个用户 |
| **总CAC** | **¥10** | |

### 计算
- 总获客成本：¥10,000
- 获取用户数：1,000人
- CAC = ¥10,000 / 1,000 = ¥10
```

### Phase 3: 计算 LTV/CAC

```markdown
## 健康度判断

| LTV/CAC | 健康度 | 说明 |
|---------|--------|------|
| < 1 | ❌ 危险 | 每用户都在亏钱 |
| 1-3 | ⚠️ 边缘 | 勉强打平，无利润空间 |
| 3-5 | ✅ 健康 | 可持续，有优化空间 |
| > 5 | 🚀 优秀 | 可加大投入 |

### 记账App现状
- LTV = ¥60
- CAC = ¥10
- LTV/CAC = 6 ✅ 优秀

**结论**：可以加大获客投入
```

### Phase 4: 计算回收周期

```markdown
## CAC回收周期

### 计算
回收周期 = CAC / 月ARPU

### 记账App
- CAC = ¥10
- 月ARPU = ¥6
- 回收周期 = 1.67个月

### 健康标准
- < 6个月：✅ 优秀
- 6-12个月：⚠️ 可接受
- > 12个月：❌ 资金压力大
```

---

## Output

```markdown
# 单位经济模型

## 核心指标
| 指标 | 数值 | 健康度 |
|------|------|--------|
| LTV | ¥60 | - |
| CAC | ¥10 | - |
| LTV/CAC | 6 | ✅ 优秀 |
| 回收周期 | 1.67月 | ✅ 优秀 |

## 敏感性分析
| 场景 | LTV/CAC | 结论 |
|------|---------|------|
| 当前 | 6 | 健康 |
| CAC上涨50% | 4 | 仍可接受 |
| 流失率翻倍 | 3 | 需优化留存 |

## 建议
- 当前模型健康，可加大获客投入
- 重点优化留存，提升LTV
```
