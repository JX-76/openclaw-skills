# Requirement Clarify - 需求澄清

---

## Description

将模糊的一句话需求转化为可执行的详细需求。

**解决的实际问题：**
- 老板/业务方只说"做个XX功能"
- 需求描述空洞（"优化体验"）
- 信息不足无法进入开发

---

## When to Use

**强制触发条件（任一满足）：**
- 用户需求 < 30 字
- 需求描述中包含"优化"、"提升"、"完善"等模糊词且无具体说明
- 用户说"做个XX功能"但无细节

**示例触发句：**
- "做个会员系统"
- "优化一下下单流程"
- "参考淘宝做个直播功能"
- "提升用户留存"

---

## Pre-conditions

**用户已提供（即使很少）：**
- 需求的一句话描述

---

## Workflow

### Phase 1: 需求解码

**提取隐含信息：**
```
用户说："做个会员系统"

隐含问题：
- 会员分级？几级？
- 权益有哪些？
- 收费还是免费？
- 对标哪个产品？
```

### Phase 2: 结构化提问

**必问问题（5W2H 精简版）：**

```markdown
## 关于用户（Who）
1. 这个功能主要给什么角色用？
   - 内部员工 / 外部用户 / 商家 / 运营

## 关于场景（When & Where）
2. 用户在什么情况下会用这个功能？
   - 触发场景是什么？

## 关于目标（Why）
3. 做了这个功能希望达到什么效果？
   - 提升什么指标？
   - 解决什么痛点？

## 关于范围（What）
4. 功能范围是？
   - MVP 必须有什么？
   - 第一期不做什么？

## 关于对标（Reference）
5. 有参考的竞品或产品吗？
   - 哪个功能点值得借鉴？
```

**提问策略：**
- 一次问 2-3 个问题（避免信息过载）
- 提供选项让用户选择（降低回答成本）
- 示例："会员等级建议 3 档（普通/白银/黄金）还是 4 档（增加钻石）？"

### Phase 3: 需求确认

**整理确认格式：**
```markdown
## 需求澄清结果

### 核心定位
- 目标用户：电商平台商家
- 核心场景：管理店铺会员，提升复购率
- 预期效果：会员复购率提升 20%

### 功能范围（MVP）
- ✅ 会员等级体系（3级：普通/白银/黄金）
- ✅ 积分累计与兑换
- ✅ 会员专属折扣
- ❌ 会员储值（二期）
- ❌ 会员营销工具（二期）

### 对标参考
- 参考：淘宝店铺会员、有赞会员系统

### 待确认问题
- [ ] 积分有效期多久？
- [ ] 升级规则是按消费金额还是订单数？
```

---

## Output

**澄清完成标准：**
```
✅ 需求已澄清，可以开始撰写 PRD

[需求确认结果摘要]

下一步建议：
使用 /write-prd 撰写详细需求文档
```

**澄清未完成：**
```
⏳ 还需补充以下信息：
1. xxx
2. xxx

[继续提问]
```

---

## Quality Gates

- [ ] 目标用户明确（非"所有用户"）
- [ ] 核心场景可描述（When/Where）
- [ ] 成功标准可量化或明确"暂不量化"
- [ ] 功能范围有边界（有什么/没有什么）
