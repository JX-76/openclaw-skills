# User Research - 用户调研设计

---

## Description

设计结构化用户调研方案，获取真实用户需求和反馈。

**解决的实际问题：**
- 需求来自"我觉得"而非"用户需要"
- 调研问题设计不当，得不到有效信息
- 调研结果无法指导产品决策

---

## When to Use

**强制触发条件（任一满足）：**
- 用户明确说"调研"、"访谈"、"问卷"
- 提到"了解用户需求"、"用户反馈"
- 产品从0到1，需要验证假设

**示例触发句：**
- "设计一个用户调研方案"
- "访谈问题怎么设计？"
- "验证用户是否愿意付费"

---

## Pre-conditions

**必须提供：**
- 调研目标（要解决什么问题）
- 目标用户群体

**可选提供：**
- 现有假设/猜测
- 可用资源（时间/预算）

---

## Workflow

### Phase 1: 调研目标澄清

**将模糊目标转化为可验证假设：**
```
模糊："了解用户购物流程"
↓
具体："用户在下单过程中，在哪个环节放弃率最高，原因是什么？"
↓
假设："50%用户在运费计算环节因运费过高而放弃"
```

### Phase 2: 选择调研方法

| 方法 | 适用场景 | 成本 | 样本量 |
|------|---------|------|--------|
| 深度访谈 | 探索性研究，了解动机 | 高 | 5-10人 |
| 问卷调查 | 验证假设，量化数据 | 低 | 100+人 |
| 可用性测试 | 验证设计方案 | 中 | 5-8人 |
| 行为数据分析 | 发现使用模式 | 低 | 全量 |

**组合建议：**
- 探索期：深度访谈 → 问卷验证
- 验证期：可用性测试 → A/B测试

### Phase 3: 设计调研内容

**深度访谈大纲示例：**
```markdown
## 开场（5分钟）
- 自我介绍，说明目的
- 获得录音许可
- 暖场：日常购物习惯

## 主体（30分钟）
### 现状了解
1. 您平时网购的频率？主要买什么东西？
2. 最近一次购物是什么？（让用户回忆具体场景）

### 痛点挖掘
3. 那次购物中，有没有遇到什么不满意的地方？
   - 追问：具体是什么？当时您怎么处理的？
4. 您觉得目前的购物流程，最麻烦的是哪个环节？

### 解决方案验证
5. 如果我们提供XX功能，您觉得有用吗？
   - 追问：什么情况下会用？愿意为此多付时间/钱吗？

## 结束（5分钟）
- 还有什么想补充的？
- 感谢并告知后续
```

**问卷设计原则：**
- 题目数 < 15（完成率 > 80%）
- 选择题为主，填空题 < 3个
- 敏感问题（收入）放在最后
- 必答题标记清楚

### Phase 4: 样本与执行

**招募标准：**
```markdown
## 目标用户画像
- 角色：电商卖家
- 年龄：25-40岁
- 经验：有1年以上网店运营经验
- 平台：淘宝/拼多多/抖音小店
- 痛点：正在使用或考虑过使用数据分析工具

## 招募渠道
- 现有用户池（如有）
- 行业社群
- 付费招募（100-200元/人）
```

---

## Output

```markdown
# 用户调研方案

## 调研目标
验证假设：用户在下单过程中，运费环节放弃率最高

## 调研方法
深度访谈（定性）+ 问卷（定量）

## 访谈大纲
[详见Phase 3]

## 样本计划
- 访谈：8人（5成功用户 + 3流失用户）
- 问卷：200人

## 执行时间表
| 阶段 | 时间 | 产出 |
|------|------|------|
| 方案确认 | D1 | 本文档 |
| 用户招募 | D1-3 | 确定访谈对象 |
| 访谈执行 | D4-7 | 访谈记录 |
| 问卷投放 | D5-7 | 回收问卷 |
| 分析总结 | D8-9 | 调研报告 |

## 成功标准
- 访谈：发现 ≥3 个共性痛点
- 问卷：回收率 > 30%，假设验证置信度 > 80%
```
