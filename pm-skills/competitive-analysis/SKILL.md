# Competitive Analysis - 竞品分析

---

## Description

结构化分析竞品，提取可借鉴功能点和差异化机会。

**解决的实际问题：**
- 竞品分析变成功能罗列，无结论
- 不知道竞品为什么这样做
- 分析完不知道对自己产品有什么用

---

## When to Use

**强制触发条件（任一满足）：**
- 用户明确说"竞品分析"、"对标"
- 提到"看看XX怎么做的"
- 产品从0到1，需要确定差异化定位

---

## Pre-conditions

**必须提供：**
- 竞品名称或领域
- 分析目标（学什么/避什么坑/找差异化）

---

## Workflow

### Phase 1: 确定分析维度

**根据目标选择维度：**

| 分析目标 | 重点维度 |
|---------|---------|
| 学习功能设计 | 功能清单、交互流程 |
| 找差异化定位 | 目标用户、核心价值、价格策略 |
| 避坑 | 用户差评、功能缺陷、运营策略 |

### Phase 2: 功能对比矩阵

```markdown
| 功能 | 我方 | 竞品A | 竞品B | 差异分析 |
|------|------|-------|-------|---------|
| 登录方式 | 手机号 | 手机号+微信 | 手机号+微信+Apple | 竞品更全 |
| 首页布局 | 列表 | 卡片 | 列表+推荐 | 卡片更直观 |
| 搜索 | 关键词 | 关键词+筛选 | 关键词+语音 | 语音是趋势 |
```

### Phase 3: 用户体验地图

**关键路径走查：**
```markdown
## 核心流程：下单

### 步骤1：浏览商品
- 我方：3步进入商品详情
- 竞品A：2步，有快速预览
- 竞品B：4步，但推荐精准

### 步骤2：加入购物车
- 我方：点击后提示成功
- 竞品A：动画飞入购物车
- 竞品B：直接跳转购物车

### 步骤3：结算
- 我方：5个字段必填
- 竞品A：3个字段，默认填充
```

### Phase 4: 用户反馈分析

**收集渠道：**
- 应用商店评论（1-3星重点看）
- 社交媒体提及
- 客服反馈

**分析维度：**
```markdown
| 问题类型 | 提及次数 | 严重程度 | 我们是否也有 |
|---------|---------|---------|-------------|
| 加载慢 | 23 | 高 | 是 |
| 退款难 | 15 | 高 | 否 |
| 搜索不准 | 8 | 中 | 是 |
```

### Phase 5: 结论与建议

```markdown
## 关键发现
1. 竞品A的语音搜索是差异化亮点，用户好评度高
2. 行业普遍痛点：退款流程复杂
3. 我方优势：推荐算法更精准

## 行动建议
- 短期：优化加载速度（影响最大）
- 中期：调研语音搜索需求
- 长期：建立退款流程优势
```
