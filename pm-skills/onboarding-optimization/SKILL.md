# Onboarding Optimization - 新用户激活

---

## Description

优化新用户首次体验，提升激活率和次日留存。

**解决的实际问题：**
- 用户下载了但不知道怎么用
- 注册完就流失，没体验到价值
- 激活流程太长，用户不耐烦

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"激活"、"onboarding"、"新用户引导"
- 新用户次日留存 < 30%
- 注册→激活转化率 < 50%
- 用户反馈"不知道怎么用"

**不适用场景：**
- 产品简单到无需引导（如计算器）
- 用户是专业人士（如开发者工具）

---

## Pre-conditions

**必须提供：**
- 当前激活率数据
- 用户流失节点（哪一步流失最多）
- 核心功能使用数据

---

## Workflow

### Phase 1: 诊断激活漏斗

```markdown
## 激活漏斗分析

### 当前漏斗
| 步骤 | 转化率 | 流失原因猜测 |
|------|--------|-------------|
| 下载App | 100% | - |
| 打开App | 70% | 启动慢/闪退 |
| 注册/登录 | 50% | 注册流程复杂 |
| 完成首单 | 30% | 不知道核心功能 |
| 次日返回 | 25% | 没感受到价值 |

### 关键问题
- 注册环节流失 20%（是否需要强制注册？）
- 首单完成率低（核心功能不突出）
```

### Phase 2: 设计激活路径

```markdown
## Aha Moment 识别

### 什么是Aha Moment？
用户首次体验到产品价值的时刻

### 记账App的Aha Moment
- 完成第一笔记账 → 3秒完成，"真快"
- 看到月度统计 → "原来我花了这么多"

### 激活路径设计
```
下载 → 启动 → 跳过注册 → 直接记账 
   → 完成 → 引导注册（保存数据）
   → 展示统计 → 推送提醒设置
```

### 关键优化点
| 节点 | 优化策略 | 预期提升 |
|------|---------|---------|
| 启动页 | 不强制注册，「先试用」 | 注册流失 -50% |
| 首单引导 | 高亮「+」按钮，动画提示 | 首单完成 +30% |
| 价值展示 | 首单后立即显示统计 | Aha时刻提前 |
```

### Phase 3: 设计引导流程

```markdown
## 引导方式选择

| 方式 | 适用 | 示例 |
|------|------|------|
| 渐进式 | 功能复杂 | 分步骤展示功能 |
| 工具提示 | 关键功能 | 高亮「+」按钮 |
| 空状态 | 无数据时 | 示例数据展示 |
| 视频 | 操作复杂 | 30秒教程 |

### 引导原则
- 不超过3步
- 可跳过
- 有明确进度
- 每一步都有价值
```
