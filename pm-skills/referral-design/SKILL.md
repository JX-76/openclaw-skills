# Referral Design - 推荐体系设计

---

## Description

设计有效的用户推荐体系，让老用户带来新用户。

**解决的实际问题：**
- 用户推荐了但没奖励，没动力
- 奖励给了但没带来好用户
- 作弊薅羊毛严重

---

## When to Use

**强制触发条件（必须满足）：**
- 产品有清晰的分享场景
- 现有用户有推荐意愿（NPS > 30）
- 可追踪推荐来源

**不适用场景：**
- NPS < 20（用户不愿推荐）
- 无法追踪邀请关系
- 产品价值未被验证

---

## Pre-conditions

**必须提供：**
- 当前NPS分数
- 用户分享动机（为什么推荐）
- 可追踪的技术方案

---

## Workflow

### Phase 1: 设计激励结构

```markdown
## 激励结构设计

### 双边 vs 单边
| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 双边 | 邀请人和被邀请人都有奖励 | 双方都需要动力 |
| 单边 | 仅邀请人/被邀请人有奖励 | 一方动力已足够 |

### 激励类型
| 类型 | 示例 | 成本 | 效果 |
|------|------|------|------|
| 现金 | ¥10/人 | 高 | 强，易薅羊毛 |
| 积分 | 100积分 | 中 | 中，需设计兑换 |
| 功能 | 解锁高级功能 | 低 | 强，精准用户 |
| 虚拟 | 徽章/等级 | 低 | 弱，长期价值 |

### 推荐方案示例
- 邀请人：获得1个月高级会员
- 被邀请人：首单8折
- 封顶：每月最多邀请10人（防作弊）
```

### Phase 2: 设计分享流程

```markdown
## 分享流程优化

### 关键触点
1. **价值峰值**（用户刚获得价值时）
   - 记账完成→生成报告→"分享给朋友"
   
2. **成就时刻**
   - 坚持记账30天→"邀请朋友一起"

3. **付费转化后**
   - 付费成功→"邀请朋友，各得¥10"

### 分享文案模板
```
标题：我发现一个超快的记账App
内容：3秒记一笔，月底自动统计
   我用了一个月，终于知道钱花哪了
   推荐你也试试
链接：[邀请链接]
```

### 落地页设计
- 显示邀请人头像（建立信任）
- 明确奖励（"朋友送你¥10"）
- 简化注册流程（一键授权）
```

### Phase 3: 防作弊机制

```markdown
## 反作弊规则

### 识别作弊信号
| 信号 | 阈值 | 动作 |
|------|------|------|
| 同一设备多账号 | >3个 | 冻结奖励 |
| 短时间大量邀请 | >10人/小时 | 人工审核 |
| 被邀请人行为异常 | 留存<1% | 不计入有效 |
| 虚拟手机号 | - | 直接拒绝 |

### 奖励发放规则
- 被邀请人完成核心行为后才发放
- 延迟发放（7天后）
- 设置每月邀请上限
```
