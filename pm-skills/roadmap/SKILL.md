# Roadmap - 产品路线图规划

---

## Description

规划产品中长期发展路径，平衡短期交付与长期目标。

**解决的实际问题：**
- 老板要3年规划，但不知道怎么做
- 需求乱排，没有逻辑主线
- 资源总是不够，优先级天天变

---

## When to Use

**强制触发条件（任一满足）：**
- 明确说"路线图"、"roadmap"、"规划"
- 需要向老板/投资人汇报规划
- 团队扩张，需要对齐方向
- 战略级项目启动

**不适用场景：**
- 单迭代的需求排期（用优先级排序）
- 临时性/探索性项目（等方向明确）

---

## Pre-conditions

**必须提供：**
- 产品愿景/目标
- 当前阶段（探索期/增长期/成熟期）
- 资源约束（团队规模/预算/时间）

**可选提供：**
- 市场/竞品信息
- 用户反馈汇总
- 技术债务情况

---

## Workflow

### Phase 1: 战略定位

```markdown
## 产品愿景
成为XX领域最XX的XX，帮助用户解决XX问题

## 当前阶段
- 阶段：增长期（已验证PMF，追求规模化）
- 核心目标：提升用户留存和商业化能力
- 关键指标：DAU从10万→50万，付费率从2%→5%

## 时间范围
- 规划周期：未来6个月
- 复盘节奏：每月review，每季度调整
```

### Phase 2: 主题聚类

**将分散需求聚合成战略主题：**
```markdown
## 战略主题（Q2-Q3）

### 主题1：用户体验升级（用户体验 → 留存）
- 核心问题：新用户流失率高（次日留存<30%）
- 关键举措：
  -  onboarding流程优化
  -  核心功能引导
  -  个性化推荐
- 成功指标：次日留存提升至40%

### 主题2：商业化探索（变现能力）
- 核心问题：付费转化低（2%），ARPU值低
- 关键举措：
  -  会员体系设计
  -  增值服务包装
  -  定价策略优化
- 成功指标：付费率提升至5%，ARPU提升50%

### 主题3：技术基建（支撑能力）
- 核心问题：性能瓶颈，迭代速度慢
- 关键举措：
  -  数据库优化
  -  微服务拆分
  -  自动化测试
- 成功指标：页面加载<1s，发布周期缩短50%
```

### Phase 3: 版本规划

```markdown
## Roadmap 时间轴

### Q2 (4-6月)
| 月份 | 主题1 | 主题2 | 主题3 |
|------|-------|-------|-------|
| 4月 | onboarding优化v1 | - | 数据库优化 |
| 5月 | 功能引导系统 | 会员体系MVP | - |
| 6月 | 个性化推荐v1 | 会员权益上线 | 微服务拆分启动 |

**里程碑：**
- 4月底：新用户留存提升至35%
- 6月底：会员系统上线，付费率达到3%

### Q3 (7-9月)
| 月份 | 主题1 | 主题2 | 主题3 |
|------|-------|-------|-------|
| 7月 | 推荐算法优化 | 增值服务设计 | 微服务完成 |
| 8月 | 用户体验闭环 | 定价A/B测试 | 自动化测试覆盖60% |
| 9月 | - | 商业化复盘优化 | - |

**里程碑：**
- 9月底：留存40%，付费率5%，技术债务清零
```

### Phase 4: 资源匹配

```markdown
## 资源分配

### 人力分配（10人团队）
| 主题 | 人数 | 占比 |
|------|------|------|
| 用户体验升级 | 4人 | 40% |
| 商业化探索 | 3人 | 30% |
| 技术基建 | 2人 | 20% |
| 机动/Bugfix | 1人 | 10% |

### 关键依赖
- 推荐算法：依赖数据团队支持
- 会员支付：依赖财务/法务合规
- 微服务：依赖运维配合

### 风险缓冲
- 每个主题预留20% buffer时间
- 6月底评估Q3计划，可调整
```

### Phase 5: 沟通与对齐

```markdown
## Roadmap 沟通

### 对老板
- 一页纸概览（主题+里程碑+资源）
- 关键决策点标注（需要支持的）
- 风险与应对

### 对团队
- 详细版本计划（具体到功能）
- 个人目标与Roadmap关联
- 月度review同步进展

### 对协作方
- 依赖项提前沟通（给对方预留时间）
- 里程碑节点主动同步
```

- [ ] 资源分配总和 ≤ 100%
- [ ] 关键依赖已识别并沟通
- [ ] 有定期复盘机制
